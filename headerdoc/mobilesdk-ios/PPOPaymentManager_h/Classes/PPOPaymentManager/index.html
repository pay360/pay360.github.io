Introduction

A PPOPaymentManager object lets you make payments, with a convenient completion block callback. Each payment session can be configured with a time limit, which ensures the payment takes a limited time to complete.

initWithBaseURL:

-(instancetype)initWithBaseURL:(NSURL*)baseURL;
Parameters
baseURL

The base URL that represents the environment for making payments.

Return Value

An instance of PPOPaymentManager

Discussion

The base URL is set once, for the lifetime of the instance and is used throughout all subsequent calls.

isSafeToRetryPaymentWithOutcome:

+(BOOL)isSafeToRetryPaymentWithOutcome:(PPOOutcome*)outcome;
Parameters
outcome

The completion handler for the payment.

Return Value

A boolean indication that it is safe to re-attempt a payment, without risking a duplicate payment.

Discussion

Determines if a payment can be re-attempted without risk of a duplicate payment being made. If there is any chance of duplciation, this method returns NO.

makePayment:withTimeOut:withCompletion:

-(void)makePayment:(PPOPayment*)payment withTimeOut:(NSTimeInterval)timeout 
        withCompletion:(void(^)(PPOOutcome *outcome))completion;
Parameters
payment

Details of the payment.

timeout

60.0 seconds is the minimum recommended value, to ensure a payment has enough time to complete. Once this timeout expires, your payment session will terminate and the completion block will fire.

completion

The outcome handler for the payment.

Discussion

Makes a payment using the supplied information.
